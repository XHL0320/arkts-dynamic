# Non-sendable Control Bytecode Candidates

## Scope

This file records bytecode expected candidates for non-sendable class control cases. These serve as comparison baselines for sendable decorator bytecode. **No real bytecode .expected.txt files are generated.** These are candidate descriptions only.

Stage 6: non-sendable control cases verify that the same decorator pattern without sendable class does NOT produce sendable-specific bytecode, confirming that the PR 10632 bug was specific to sendable + decorator interaction.

## Candidate 1: non-sendable property decorator factory argument

**Description:** Bytecode expected for a non-sendable class with a property decorator factory whose argument is an IIFE capturing a local binding. This is the control comparison for PR 10632.

**Expected check points (NOT real expected output):**

1. Class `A` bytecode does NOT contain sendable class markers
2. Constructor bytecode is ordinary (no `"use sendable"`)
3. IIFE argument function bytecode does NOT contain InSendable flag
4. Decorator factory return function bytecode does NOT contain InSendable flag
5. ALL functions in this file have inSendable=false in bytecode
6. No SendableFunction markers anywhere in generated bytecode
7. Compare with sendable variant (Candidate 1 in pr10632_bytecode_candidates.md):
   - Only difference should be class body markers (sendable vs non-sendable)
   - All decorator-related functions should have identical inSendable=false
